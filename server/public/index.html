Hello, Everyone

In this project we can experiment with HTTP verbs, Docker, MongoDB, and React.js.
The project purpose is to demonstrate a mastery of the network protocols and storage capabilities. The architecture of this project is as follows. The main component of this architecture is express. Express is our master server. Where we connect to the database and prepare to recieve HTTP requests. We can send data through our express rest-api with all http verbs supported. These parameters sent through the network are then stored in our mongodb database. From our mongodb, we can get all legitamite parameters because mongodb's schemas have validation that only allow the desired state of the variables. Our next component is our front-end; react. React recieves all data from our rest-api and no outside sources. React also has forms available to populate our database with more indeces. But we don't access the database directly. Instead through react we send a post request to our express server. Validating our parameters and inserting into our collection.

Docker

Docker is a containerization client. Whose purpose is to encapsulate your built projects and give them a container environment to run on. This webpage is connected in series with two other containers running together. This project can also be deployed in a swarm architecture because of the docker-compose.yaml.

Nginx

We use it in this project as a load balancer for our rest-api running on port 5000. We want nginx as a load balancer because we want to be prepared to recieve multiple incoming requests at the same time! This is a way we can prepare to scale the data recieved through our rest.
